This is to summarize  steps needed initially to run the py code in Granite

- Input is config-settings.sx file to the code. The code will read the contents of the file and checks the values against the config.db in the phone
- You can extract the config-settings.sx file in VBR >> ConE output >> ... >> varinat0 >> config-settings.sx (locaiton might be different)
- Copy the config-settings.sx file to the customization folde, where the code is
- Copy the customization folder to the Granite >> testscripts folder
- Copy the testsets >> customizationtest.testset to the Granite >> testsets folder
